10原型交付包是什么

原型交付包是什么

一个标准需求目录里，主要就两类文件：

  • PRD 文件（*.md）— 需求来源与规则说明
  • 原型文件（prototype/）— 可 preview 的 HTML 原型交付包
需求目录/
├── feature.md
└── prototype/
├── index.html
└── prototype-content/
11目录是讲解路线

目录是讲解路线

目录不是网站导航，而是按章节推进的讲解路线。多个独立原型界面不会散掉，使用者可以按章节讲。

1 · 开场
2 · 生成能力
3 · 宣讲解决方案
4 · 本地修订方案
12PRD Viewer 是需求依据

PRD Viewer 是需求依据

PRD Viewer 承载长说明、规则、背景和验收标准；原型 screen 承载用户真实能看到的界面。二者并排帮助解释「为什么生成成这样」。

原型 Screen
PRD Viewer
长说明归位：规则、背景和验收标准不要塞进 screen。
13标注与 Spotlight 是讲解辅助

标注与 Spotlight 是讲解辅助

  • Annotation：短规则、风险、待确认点
  • Spotlight：聚焦当前界面，放大查看细节
  • 演讲光标：指向关键区域
界面主 Surface
Annotation 区
Spotlight 区
标注使用 skill 原生样式，放在 surface 外侧。

  • 操作：进入「改文字」模式，修改安全文本并保存

15大改备注给 AI

大改备注给 AI

结构、流程、布局、状态变化不在页面里硬改。使用者点选目标元素，写下修改意图；备注携带 data-proto-id / data-proto-label，方便下一轮 AI 定位。

16质量检查概念

质量检查概念

ProtoPilot 生成物不是「画完就算」，它有基本的交付检查闭环。

✓ 内容包完整
✓ 计划与页面一致
✓ PRD 可读取
✓ 目录可导航
✓ 标注位置安全
✓ 预览可打开

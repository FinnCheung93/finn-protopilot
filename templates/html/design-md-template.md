# Design 文件夹模板

初始化产品级 `Design/` 文件夹时使用这些模板。

## Design/README.md

```markdown
# Design 目录说明

本目录为本产品的 ProtoPilot 原型生成上下文，用于沉淀项目级视觉、交互、参考图和可复用素材。

## 文件与目录

- `design.md`：项目级视觉与交互规则。写品牌色、字体、页面密度、常见组件、平台差异、参考图索引。
- `references/`：项目通用参考图。放已有 App/Web 页面截图、典型组件截图、历史原型截图。
- `assets/`：项目通用素材。放 Logo、通用插画、少量高频图片。

## 不放什么

- 不放单个需求的一次性临时图；放到对应需求文件夹的 `references/`。
- 不放正式 UI 源文件大包；只放原型生成真正会用到的轻量图片。
- 不放研发实现资料、接口文档或埋点表。

## 命名建议

- 使用可读英文或拼音短名，例如 `message-center-list.png`、`permission-card.png`。
- 避免 `截图1.png`、`image.png`、`未命名.png`。

## 使用原则

- ProtoPilot 生成原型时优先读取 `design.md`。
- 参考图用于视觉对齐和布局理解，不作为正式 UI 验收标准。
- 正式视觉仍以 UI 设计稿为准。
```

## Design/design.md

```markdown
# ProtoPilot Design Context

## 1. 产品定位

- 产品名：<产品名>
- 主要平台：Mobile App / Web Page / Desktop Page
- 原型默认用途：PRD 宣讲、需求走查、开发/UI 理解

## 2. 视觉基础

- 主色：<如 #1677FF>
- 背景色：<如 #F7F9FC>
- 页面 surface：<如 #FFFFFF>
- 主文字：<如 #111827>
- 弱文字：<如 #6B7280>
- 圆角：<如 12px / 16px>
- 阴影：轻量即可，避免重拟物

## 3. 平台展示规则

- Mobile App：使用手机框展示。
- Web Page：直接使用大屏页面区域，不套拟真浏览器框。
- Desktop Page：直接使用大屏页面区域，不套桌面设备框。
- Mixed flow：按每个 Step 的平台选择展示方式。

## 4. 组件倾向

- 图标：默认 Lucide。
- 按钮：主按钮突出，次按钮弱化。
- 列表：清晰分组，避免过密。
- Toast/Dialog：只用于关键反馈，不滥用。
- 标注：只标关键入口、状态变化、异常提示和核心业务规则。

## 5. 参考图索引

| 场景 | 参考图 | 说明 |
| --- | --- | --- |
| <页面/模块> | `references/<file>.png` | <要参考的布局或视觉点> |

## 6. 素材索引

| 素材 | 路径 | 用途 |
| --- | --- | --- |
| <素材名> | `assets/<file>.png` | <用途> |

## 7. 原型边界

- 原型服务 PRD 宣讲，不代表最终 UI。
- 未提供真实素材时，使用 Lucide、CSS 形状、简洁占位。
- 长规则和长表格应放在说明面板或标注中，不塞进手机屏。
```
